## 本次目标
v0.3.x：代码结构优化 + 控制算法优化 + 串口控制实现。

### 优先级与阶段划分

#### P0（功能实现）
P0.1 协议解析（新增 L3 文件）
- 目标：建立轻量文本协议的帧解析与结构化命令缓存。
- 范围：只做“收帧、解析、缓存、取数、异常回报”，不在本模块执行命令。
- 帧格式（固定约束）：帧头-命令-子命令-参数-帧尾，示例 `aCD1.234b`。
- 接口要求：
	- 提供输入触发接口（中断回调置位）与轮询解析入口（主循环/任务调用）。
	- 提供底层读取适配接口，使用外设层读取 API 获取接收缓存。
	- 提供解析结果读取接口（含“是否更新/是否脏数据”标志）。
	- 提供异常回报接口调用（走特殊依赖层，单字符短回报）。
- 数据要求：
	- 结构化存储命令、子命令、参数字符串/数值。
	- 缓存最近一次有效指令与状态（成功/帧错/超时）。
	- 帧头/帧尾使用可配置宏（默认 `a` / `b`）。
	- 一条命令仅允许一个参数修改。
	- 数值解析：支持带符号十进制浮点；不支持科学计数法。
	- 解析路径应优先降低时间复杂度，减少不必要判断与拷贝。
- 验收标准：可接收帧、可读取结构化结果、可回报帧有效性。

P0.2 命令逻辑 + 状态参数管理 + 诊断信息（新增 L2 文件）
- 目标：统一命令执行入口、状态读写入口、初始化诊断信息输出。
- 职责边界：
	- L3 协议层只产生命令对象。
	- L2 命令层负责命令分发、参数更新、状态变更。
- 交付内容：
	- 统一状态结构体与读写 API（初始化/运行/故障/通信/调试模式）。
	- 启动诊断模板（成功/失败/未执行）与最小回退策略。
	- 参数管理接口（读单参数、读全参数、写参数）与修改标志。
	- 参数读取输出统一为文本语义化输出（读单参数/读全参数一致）。
- 约束：与算法低耦合；诊断能力可宏裁剪。
- 运行约束：命令权限不区分运行态/停止态。
- 验收标准：上电日志可定位到模块级失败原因；状态不再由散落全局变量拼装。

P0.3 调试输出体系完善（低频语义化 + 高频示波器）
- 目标：建立分类、可开关、可分频的双通道调试输出。
- 输出策略：
	- 低频通道：语义化文本（面向人读）。
	- 高频通道：示波器数据（二进制帧头/帧尾 + 文本参数混合）。
	- 支持可配置输出频率、字段数量、字段顺序。
- 接口要求：编译期开关 + 运行期开关预留。
- 验收标准：各类输出可独立启停，且不影响主控制节拍。

P0.4 外设层串口接口（L4）
- 目标：为上层协议提供稳定吞吐输入输出能力。
- 约束：USART 发送 DMA；接收 DMA + IDLE 中断；接收采用双缓冲策略。
- 交付内容：
	- 接收触发接口（IDLE 事件/回调）。
	- 接收缓存读取接口（长度、数据地址或复制接口）。
	- 发送接口（字符串/二进制）。
- 验收标准：协议层可稳定触发解析，长短帧均可读。

### P0 依赖顺序
1. 先完成 P0.4（串口基础输入输出）。
2. 再完成 P0.1（协议解析与结构化缓存）。
3. 然后完成 P0.2（命令执行与状态管理）。
4. 最后完成 P0.3（调试输出完善与联调）。

### P0 已确认协议约束（用于开发实现）
1. 帧头/帧尾：可配置宏，默认 `a` / `b`。
2. 单命令修改：每条命令仅允许一个参数修改。
3. 数值格式：支持带符号十进制浮点，不支持科学计数法。
4. 异常回报：按单字符编码实现（默认 `E` 帧错、`T` 超时、`O` 正常），并提供宏可变更。
5. 接收策略：DMA + IDLE + 双缓冲。
6. 参数读取：读单参数和读全参数均采用文本语义化输出。
7. 命令权限：不区分运行态/停止态。
8. 性能要求：解析路径尽量降低时间复杂度，减少无效判断。
