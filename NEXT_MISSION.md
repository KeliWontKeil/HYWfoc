## v1.8.2 规划

- P0：电流采样链路评估与整理
  - 目标：先厘清快/慢电流采样的真实差异和必要性，再简化采样相关函数
  - 现状关注点：当前慢采样与快采样在 PAL 层主要区别是 ADC 平均窗口；慢窗口配置意图是覆盖 1 个控制周期，但当前实例 ADC DMA buffer 较小，实际平均窗口可能被底层截断，需先按代码确认真实行为。
  - 拆分：
    1. 梳理 `Sensor_ReadAll`、`Sensor_ReadCurrentOnly`、`Sensor_ReadADC`、`FOC_Platform_ReadPhaseCurrent`、`FOC_Platform_ReadPhaseCurrentFast`、`ADC_ReadPhaseCurrentABOk` 的调用链和执行上下文。
    2. 明确快/慢采样是否都需要保留：若只是平均窗口差异，优先收敛为内部统一读取函数，并用采样 profile 或 avg_count 表达差异。
    3. 处理 `sensor_data` 静态全局在控制任务和 PWM ISR 路径同时更新的风险，避免读写半更新快照。
    4. 修正与代码不一致的注释、配置含义或文档描述。
  - 验收：
    - 电流采样入口语义清晰，控制环路径和电流环 ISR 路径职责明确。
    - 构建 0 error，且不新增 warning。

- P3：位置传感器采样位置选择
  - 目标：允许用户配置外部位置传感器读取位置：控制环采样用于慢速/阻塞式传感器，电流环或快速路径采样用于快速/非阻塞式传感器。
  - 实施前提：先完成电流采样链路整理，明确快路径中哪些传感器 API 可以安全调用。
  - 设计边界：
    - 当前 `FOC_Platform_ReadMechanicalAngleRad` 未声明 ISR-safe，不能直接默认放入 PWM ISR。
    - 快路径采样只应支持非阻塞传感器，如硬件捕获、霍尔状态缓存、DMA/定时器锁存结果；I2C/SPI 阻塞读取应保留在控制环或服务任务。
  - 拆分：
    1. 新增位置传感器采样位置配置项：控制环采样 / 快速路径采样。
    2. 将角度读取与电流读取解耦，避免 `Sensor_ReadAll` 成为唯一全量入口。
    3. 若支持快速路径，新增或约束平台 API：必须非阻塞、ISR-safe，并能返回最近锁存值。
    4. 在控制算法入口明确使用哪一路角度快照，并处理无效数据降级策略。
  - 验收：
    - 默认配置保持现有行为：控制环读取位置传感器。
    - 慢速位置传感器不会被放入 ISR 路径。
    - 快速采样配置有编译期或运行期保护，避免误用阻塞式平台实现。
    - 构建 0 error，且不新增 warning。
    - 硬件验证项：慢速磁编码器、快速霍尔/缓存式传感器分别验证角度有效性和控制稳定性。

- 宏裁剪评估
